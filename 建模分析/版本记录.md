## 查看历史版本的方法

查看 v1 版本的题目分析报告：

```bash
git show 4d64033:建模分析/题目分析报告.md
```

将 v1 版本导出为一个单独文件：

```bash
git show 4d64033:建模分析/题目分析报告.md > 建模分析/题目分析报告_v1.md
```

比较当前版本和 v1 的差异：

```bash
git diff 4d64033 -- 建模分析/题目分析报告.md
```

查看提交历史：

```bash
git log --oneline --decorate
```

## 后续版本控制规则

1. 每完成一轮稳定建模修改，就提交一次 Git commit。
2. 每次提交信息写清楚本轮修改主题，例如 `Refine UAV modeling framework v2`。
3. 重要阶段可以打标签，例如 `analysis-v1`、`analysis-v2`。
4. 不提交 `.agents/`、Office 临时锁文件、格式分析中间文件和无关删除状态。
5. 如果需要保留可直接打开的历史副本，再从 Git 历史导出为 `题目分析报告_vX.md`，不要手工复制未记录版本。
